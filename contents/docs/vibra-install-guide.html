Vibra++ Plasmoid

A Shazam-like music recognition plasmoid for KDE Plasma 6

Prerequisites

    cmake \
    extra-cmake-modules \
    gcc-c++ \
    qt6-qtbase-devel \
    qt6-qtdeclarative-devel \

    plasma-devel \
    pipewire-utils \
    pulseaudio-utils \

Installation

1

Copy the plasmoid folder

Copy org.kde.plasma.vibra to your local plasmoids directory:

~/.local/share/plasma/plasmoids/org.kde.plasma.vibra/
├── contents/
├── plugin/
├── vibra/
├── CMakeLists.txt
├── build.sh
└── metadata.json
2

Set permissions

cd ~/.local/share/plasma/plasmoids/org.kde.plasma.vibra
chmod +x build.sh
find . -name "*.sh" -exec chmod +x {} \;
chmod -R a+rX .
3

Build and install

rm -rf build && ./build.sh

The script will automatically clone the vibra backend, build the plugin, and install to the correct Qt6 QML path (with sudo).

4

Restart Plasma

kquitapp6 plasmashell && kstart plasmashell
5

Add the widget

Right-click the panel → Add Widgets → search "Vibra" → add it to your panel.

First-time Setup

Open the plasmoid and go to the Settings tab:

  • Source — select the audio source:
    • Monitor of [device] — captures audio playing through that device on this computer
    • [device] input — captures ambient sound via microphone
  • Listen duration — minimum 15s, recommended 20-25s
  • Show notifications — shows a notification when minimized and a track is found
  • Start listening at startup — auto-starts when Plasma loads
  • Cover art quality — High quality (800x800) or Standard (400x400)

Troubleshooting

"vibra binary not found"

The vibra backend wasn't built. Run ./build.sh again from the plasmoid root.

"module org.kde.plasma.vibra is not installed"

Install manually to the system Qt6 QML path:

Fedora

sudo mkdir -p /usr/lib64/qt6/qml/org/kde/plasma/vibra
sudo cp ~/.local/share/plasma/plasmoids/org.kde.plasma.vibra/contents/imports/org/kde/plasma/vibra/* \
        /usr/lib64/qt6/qml/org/kde/plasma/vibra/

Arch / Ubuntu

sudo mkdir -p /usr/lib/qt6/qml/org/kde/plasma/vibra
sudo cp ~/.local/share/plasma/plasmoids/org.kde.plasma.vibra/contents/imports/org/kde/plasma/vibra/* \
        /usr/lib/qt6/qml/org/kde/plasma/vibra/

"cmake: command not found"

# Fedora
sudo dnf install cmake

# Arch
sudo pacman -S cmake

# Ubuntu
sudo apt install cmake

Recognition fails immediately

  • Check pw-record is available: which pw-record
  • Check pactl is available: which pactl
  • Check PipeWire is running: systemctl --user status pipewire
  • Try a longer listen duration (25-30 seconds)

No devices in the source dropdown

Click the refresh button next to the source dropdown. If still empty, check that pactl is installed.

Note: Settings and history survive Plasma restarts. History is limited to 50 entries. The plasmoid continues listening even when minimized.
